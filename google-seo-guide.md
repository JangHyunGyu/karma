# SEO 기본 가이드: Google 검색 스타터 가이드

> 출처: https://developers.google.com/search/docs/fundamentals/seo-starter-guide?hl=ko

## 개요

SEO(검색엔진 최적화)는 "검색엔진이 콘텐츠를 이해하도록 돕고, 사용자가 사이트를 찾도록" 하는 과정입니다. Google의 검색 Essentials를 준수하면 검색결과에 표시될 가능성이 높아집니다.

---

## Google 검색 작동 방식

Google은 자동화된 검색엔진으로 크롤러를 사용하여 웹을 탐색합니다. 대부분의 사이트는 웹마스터의 추가 조치 없이도 Google이 자동으로 발견합니다.

### 변화 반영 시간

변경사항이 검색결과에 반영되려면 수 시간에서 수 개월이 걸릴 수 있습니다. 일반적으로 효과 판정에는 수 주를 기다리는 것이 좋습니다.

---

## Google이 콘텐츠를 찾을 수 있도록 하기

### 검색 가능성 확인

`site:` 검색 연산자를 사용하여 사이트가 이미 색인되었는지 확인하세요.

```
site:wikipedia.org
```

### 페이지 인식 확인

Google이 사용자와 같은 방식으로 페이지를 인식하려면 CSS, JavaScript 등 중요한 구성요소에 접근할 수 있어야 합니다. Search Console의 URL 검사 도구를 활용하세요.

### 검색결과에서 콘텐츠 제외

- `robots.txt` 사용
- `noindex` 메타 태그 적용
- 콘텐츠 차단 설정

---

## 사이트 구성하기

### 설명적 URL 사용

사용자가 URL을 통해 콘텐츠를 파악할 수 있도록 작성합니다.

**좋은 예:**
```
https://www.example.com/pets/cats.html
```

**나쁜 예:**
```
https://www.example.com/2/6772756D707920636174
```

Google은 URL의 단어로 자동 탐색경로를 학습하며, 구조화된 데이터로 추가 제어가 가능합니다.

### 디렉터리로 주제 그룹화

비슷한 주제를 같은 디렉터리에 배치하여 Google이 업데이트 빈도를 파악하게 도와줍니다.

- `/policies/` — 자주 변경 없음
- `/promotions/` — 빈번한 변경

### 중복 콘텐츠 축소

**중복 콘텐츠:** 동일한 콘텐츠가 여러 URL에서 접근 가능한 상태

**해결 방법:**

1. 301 리디렉션 설정
2. `rel="canonical"` 링크 요소 사용
3. Google의 자동 표준화에 의존

---

## 매력적이고 유용한 콘텐츠 만들기

### 핵심 속성

#### 1. 읽기 쉬운 텍스트

- 자연스럽고 잘 작성된 콘텐츠
- 철저한 철자 및 문법 검수
- 긴 내용을 단락, 섹션으로 분리
- 명확한 제목 제공

#### 2. 고유한 콘텐츠

다른 사용자의 콘텐츠를 복사하지 말고, 본인의 지식으로 새로운 콘텐츠를 작성하세요.

#### 3. 최신 콘텐츠

기존 콘텐츠를 정기적으로 검토하고 필요시 업데이트하거나 삭제합니다.

#### 4. 유용하고 신뢰할 수 있는 인간 중심 콘텐츠

- 독자에게 가치 있는 정보 제공
- 전문성 출처 명시
- 경험 기반 작성

### 독자의 검색어 예상하기

사용자가 다양한 검색 키워드를 사용할 수 있음을 인식하세요.

> 예: '샤퀴테리' vs '가공육' (같은 주제, 다른 용어)

Google의 언어 일치 시스템이 정확한 표현 없이도 관련성을 파악합니다.

### 방해 광고 제거

광고가 콘텐츠 읽기를 방해하지 않도록 주의하세요. 특히 인터스티셜 페이지를 피하십시오.

### 관련 리소스 링크하기

#### 좋은 링크 텍스트 작성

링크 텍스트(앵커 텍스트)가 목표 페이지의 내용을 명확히 설명해야 합니다.

#### 신뢰할 수 있는 리소스만 연결

신뢰할 수 없는 외부 콘텐츠에 연결할 때는 `nofollow` 속성을 추가하세요.

#### 사용자 생성 콘텐츠 관리

포럼, 댓글의 모든 링크에 CMS에서 자동으로 `nofollow`를 추가하여 스팸을 방지합니다.

## Google 검색 결과에 표시되는 방식 최적화

### 제목 링크에 영향 주기

**제목 링크란:** 검색 결과의 헤드라인 부분으로, 사용자가 클릭 여부를 결정하는 데 영향을 줍니다.

**소스:**

- `<title>` 요소 (제목 텍스트)
- 페이지의 다른 제목

**좋은 제목 작성 팁:**

- 페이지 콘텐츠를 정확하게 설명
- 웹사이트/비즈니스 이름 포함 가능
- 위치, 서비스 정보 등 중요 정보 포함
- 명확하고 간결하게

> CMS 사용자: 대부분의 CMS가 작성한 제목을 자동으로 `<title>` 요소로 변환합니다.

### 스니펫 관리하기

**스니펫:** 제목 링크 아래의 페이지 설명으로, 클릭 여부 결정에 영향을 줍니다.

**소스:**

- 페이지 실제 콘텐츠
- 메타 설명 태그 (선택적)

**메타 설명 태그 작성법:**

- 페이지의 간결한 한두 문장 요약
- 짧고 구체적
- 해당 페이지에 고유해야 함
- 가장 관련 높은 요소 포함

## 이미지 추가 및 최적화

### 고화질 이미지 활용

고해상도 이미지로 사용자에게 충분한 맥락과 세부정보를 제공합니다.

### 관련 텍스트 근처 배치

이미지 근처의 텍스트가 Google이 이미지의 의미를 이해하는 데 도움을 줍니다.

### 설명 대체 텍스트 추가

대체 텍스트는 이미지와 콘텐츠 간의 관계를 설명하는 짧은 텍스트입니다.

**작성 방법:**

- HTML `img` 요소의 `alt` 속성 사용
- CMS에서 이미지 설명 필드 활용

> Google이 이미지 내용과 페이지와의 관계를 이해하는 데 필수적입니다.

## 무시해야 할 사항 (흔한 오해)

### 1. 메타 키워드

Google 검색은 키워드 메타 태그를 **사용하지 않습니다.**

### 2. 유인 키워드 반복

같은 단어 과도 반복은 스팸 정책 위반이며 사용자 만족도를 떨어뜨립니다.

### 3. 도메인 이름/URL 경로의 키워드

- 비즈니스에 가장 적합한 이름 선택
- 순위에 미치는 영향 미미 (탐색경로 제외)
- TLD는 지역 타겟팅 시에만 중요

### 4. 최소/최대 콘텐츠 길이

콘텐츠 길이 자체는 순위에 영향 없습니다 (최소 1단어 이상 필수).

### 5. 하위 도메인 vs 하위 디렉터리

비즈니스 관점에서 관리 용이성을 고려하여 선택하세요.

### 6. PageRank

Google의 기본 알고리즘 중 하나이지만, 순위 결정 신호는 매우 많습니다.

### 7. 중복 콘텐츠 '페널티'

- 여러 URL로 접근 가능한 콘텐츠는 무방
- 중복 콘텐츠 자체로 직접 조치 없음
- 표절은 별개의 문제

### 8. 제목 개수 및 배치 순서

- 의미론적 순서는 스크린 리더에 중요
- Google 검색 관점에서는 순서 변경 무방
- 이상적인 제목 개수 없음 (과다 시 감소 권장)

### 9. E-E-A-T를 순위 지정 요인으로 인식

Google은 E-E-A-T를 직접적인 순위 지정 요인으로 사용하지 않습니다.

# Google 검색의 작동 방식

> 출처: https://developers.google.com/search/docs/fundamentals/how-search-works?hl=ko

## 개요

이 문서는 웹사이트 환경에서 Google 검색이 어떻게 작동하는지를 단계별로 설명합니다. Google은 완전히 자동화된 검색엔진으로 웹 크롤러를 사용하여 페이지를 찾고 색인화합니다.

### 중요 사항

- Google은 금전적 대가를 받고 특정 사이트를 더 자주 크롤링하거나 순위를 높이지 않습니다.
- 페이지가 Google 검색 Essentials를 준수하더라도 크롤링, 색인 생성, 게재가 보장되지 않습니다.
- Google 검색 결과는 프로그래매틱 방식으로 결정됩니다.

---

## Google 검색의 3단계

### 1단계: 크롤링

Google은 크롤러라는 자동화된 프로그램을 사용하여 인터넷에서 찾은 페이지로부터 텍스트, 이미지, 동영상을 다운로드합니다.

#### URL 검색 프로세스

- Google이 방문한 적이 있는 페이지는 파악된 페이지 목록에 추가됨
- 새 페이지는 기존 페이지의 링크를 통해 발견됨
- 허브 페이지(카테고리 페이지)에서 새 콘텐츠로의 링크 추출
- 사이트 소유자가 제출한 사이트맵을 통해 발견되는 페이지도 있음

#### Googlebot의 역할

- 크롤링할 사이트와 빈도 결정
- 각 사이트에서 가져올 페이지 수 결정
- 과부하를 피하기 위해 사이트의 응답을 모니터링하여 크롤링 속도 조절

#### 크롤링 렌더링

- 최신 버전의 Chrome을 사용하여 페이지를 렌더링하고 발견된 자바스크립트를 실행
- 자바스크립트로 콘텐츠를 표시하는 웹사이트의 경우 렌더링이 중요

#### 크롤링이 불가능한 경우

- 서버 관련 문제
- 네트워크 문제
- robots.txt 규칙으로 인한 접근 차단
- 로그인이 필요한 페이지

---

### 2단계: 색인 생성

페이지가 크롤링된 후 Google은 페이지의 내용을 파악하고 대규모 데이터베이스(Google 색인)에 정보를 저장합니다.

#### 색인 생성 프로세스

- 텍스트, 이미지, 동영상 파일 분석
- `<title>` 요소, Alt 속성 등 핵심 콘텐츠 태그와 속성 처리
- 페이지가 인터넷에 있는 다른 페이지와 중복되는지 아니면 표준 페이지인지 판단

#### URL 표준화(Canonicalization)

- Google은 비슷한 콘텐츠의 페이지를 그룹으로 묶음(클러스터링)
- 각 클러스터에서 가장 잘 대표하는 페이지를 표준 페이지로 선택
- 그룹의 나머지 페이지는 특정 컨텍스트에서 게재 가능한 대체 버전

#### 수집되는 신호

- 페이지의 언어
- 콘텐츠가 속하는 국가
- 페이지의 사용성

#### 색인 생성이 안 되는 일반적인 이유

- 페이지 콘텐츠의 품질이 낮음
- Robots meta 규칙이 색인 생성을 허용하지 않음
- 웹사이트 디자인으로 인한 색인 생성 어려움

---

### 3단계: 검색결과 게재

사용자가 검색어를 입력하면 Google 컴퓨터는 색인에서 일치하는 페이지를 검색하고 품질과 관련성이 가장 높은 결과를 반환합니다.

#### 관련성 결정 요소

- 사용자의 위치
- 사용자의 언어
- 기기 유형(데스크톱 또는 휴대전화)
- 검색어와의 연관성

#### 지역 맞춤 검색결과

예시: '자전거 수리점'을 검색하면 파리에 있는 사용자와 홍콩에 있는 사용자에게 서로 다른 결과가 표시됩니다.

#### 색인되었으나 검색결과에 미표시 이유

- 페이지의 콘텐츠가 사용자의 검색어와 관련이 없음
- 콘텐츠 품질이 낮음
- Robots meta 규칙으로 게재가 차단됨

# 유용하고 신뢰할 수 있는 사용자 중심 콘텐츠 제작하기

> 출처: https://developers.google.com/search/docs/fundamentals/creating-helpful-content?hl=ko

## 개요

Google의 자동 순위 시스템은 검색엔진 순위를 조작하기 위해 제작된 콘텐츠가 아닌, 사람들에게 도움을 주기 위해 제작된 유용하고 신뢰할 수 있는 정보를 우선적으로 표시하도록 설계되었습니다.

---

## 콘텐츠 자체 평가하기

사이트와 관련 없는 신뢰할 수 있는 사람에게 솔직한 평가를 받을 것을 권장합니다.

### 콘텐츠 및 품질 관련 질문

- 콘텐츠에서 고유한 정보, 보고, 조사, 분석 자료를 제공하는가?
- 주제와 관련하여 본질적이거나 완전하거나 포괄적인 설명을 제공하는가?
- 뻔하지 않은 유용한 분석이나 흥미로운 정보를 제공하는가?
- 다른 출처를 활용할 때 중요한 가치와 독창성을 더했는가?
- 주요 제목이나 페이지 제목이 과장되거나 충격적이지 않은가?
- 북마크 추가, 친구와 공유, 추천할 정도의 페이지인가?
- 인쇄된 잡지, 백과사전, 도서에서 볼 수 있을 콘텐츠인가?
- 검색결과의 다른 페이지와 비교할 때 상당한 가치를 제공하는가?
- 철자 또는 스타일 문제가 있는가?
- 완성도 높게 제작되었는가, 아니면 엉성하거나 급하게 제작된 것으로 보이는가?
- 많은 제작자에 의해 대량 생산되었거나 아웃소싱되었는가?

### 전문성 관련 질문

- 명확한 출처 공개와 저자 정보를 신뢰할 수 있는 방식으로 제공하는가?
- 사이트를 살펴본 사람이 관련 주제의 권위자로 신뢰할 수 있다는 인상을 받을 수 있는가?
- 주제를 명확히 잘 아는 전문가 또는 애호가가 작성하거나 검토했는가?
- 쉽게 검증할 수 있는 사실 정보 오류가 있는가?

---

## 우수한 페이지 경험 제공

Google의 핵심 순위 시스템은 우수한 페이지 경험을 제공하는 콘텐츠에 보상을 제공합니다. 페이지 경험의 한두 가지 측면에만 몰입하기보다는 전반적으로 다양한 측면에서 훌륭한 페이지 경험을 제공해야 합니다.

---

## 사용자 중심 콘텐츠에 집중하기

### 사용자 중심 콘텐츠 확인 질문

- 콘텐츠가 유용하다고 여길 기존 또는 의도된 잠재고객층이 있는가?
- 콘텐츠에 경험을 통해 얻은 전문성과 깊이 있는 지식이 명확하게 드러나는가?
- 사이트에 주요 목적이나 초점이 있는가?
- 사용자가 콘텐츠 읽은 후 주제에 관한 충분한 지식을 학습했다고 느낄 수 있는가?
- 콘텐츠를 읽는 사람이 만족스러운 경험을 느낄 수 있는가?

---

## 검색엔진 중심 콘텐츠 만들지 않기

다음 질문 중 몇 가지에 '예'라고 답한다면 콘텐츠 제작 방식을 재평가해야 합니다.

### 피해야 할 관행들

- 주로 검색엔진에서 방문을 유도하는 콘텐츠를 제작하는가?
- 검색결과에서 좋은 실적을 올리기 위해 여러 주제의 대량 콘텐츠를 제작하는가?
- 광범위한 자동화를 사용해 다양한 주제의 콘텐츠를 제작하는가?
- 다른 사람의 이야기를 주로 요약하며 가치를 크게 더하지 않는가?
- 인기 있는 주제에만 글을 쓰고 기존 잠재고객층을 위한 글은 작성하지 않는가?
- 콘텐츠를 읽은 독자에게 다시 검색해서 다른 출처에서 더 좋은 정보를 얻어야겠다는 느낌을 주는가?
- Google에서 선호하는 특정 단어 수로 콘텐츠를 작성하는가? (Google은 특별히 선호하는 단어 수가 없음)
- 전문적인 지식 없이 틈새 주제 영역에 진출했으며 주된 이유가 검색 트래픽 확보인가?
- 확인되지 않은 정보를 제시하는가?
- 콘텐츠가 크게 변경되지 않아도 최신 페이지처럼 보이도록 페이지 날짜를 변경하는가?

### SEO에 대한 명확한 입장

검색엔진 최적화는 검색엔진 우선 콘텐츠가 아닌 사용자 중심 콘텐츠에 적용할 때 유용한 활동입니다.

---

## E-E-A-T 및 품질 평가자 가이드라인

### E-E-A-T 개념

Google의 자동화 시스템은 콘텐츠가 가진 네 가지 덕목을 평가합니다:

- **경험(Experience)**: 제품이나 서비스를 실제로 사용하거나 장소를 방문한 경험
- **전문성(Expertise)**: 주제에 관한 깊이 있는 지식
- **권위(Authority)**: 관련 분야에서 널리 인정받는 신뢰성
- **신뢰성(Trustworthiness)**: 가장 중요한 요소로, 나머지 덕목들이 이에 기여

콘텐츠가 모든 덕목을 갖춰야 하는 것은 아니며, 신뢰성이 가장 중요합니다.

### YMYL 주제의 중요성

Google 시스템은 사람들의 건강, 금융 안정성, 안전 또는 사회의 안녕, 웰빙에 큰 영향을 미칠 수 있는 주제에 대해 높은 E-E-A-T를 보여주는 콘텐츠에 더 많은 가중치를 부여합니다. 이를 YMYL(Your Money or Your Life)이라고 부릅니다.

### 검색 품질 평가자

검색 품질 평가자는 Google의 알고리즘이 신뢰할 수 있는 결과를 제공하는지에 관한 정보를 제공하는 사람들입니다.

**중요**: 검색 평가자는 페이지의 순위가 결정되는 방식에 어떤 영향도 미치지 않습니다. 평가자 데이터는 순위 산정 알고리즘에서 직접 사용되지 않습니다.

## 콘텐츠에서 '누가, 어떻게, 왜'에 관해 질문하기

### 누가 콘텐츠를 만들었는가(Who)

- 방문자가 누가 콘텐츠를 작성했는지 명확하게 알 수 있는가?
- 페이지에 작성자를 예측할 수 있게 해 주는 바이라인이 있는가?
- 바이라인에서 작성자의 배경 및 주로 다루는 분야에 관한 추가 정보를 제공하는가?

> 권장: 콘텐츠 바이라인과 같이 독자가 작성자의 정보가 있을 것이라 예상하는 장소에 정확한 저자 정보를 추가

### 어떻게 콘텐츠가 만들어졌는가(How)

- AI 생성 등 자동화를 사용한다는 사실이 방문자에게 명확히 드러나는가?
- 자동화 또는 AI 생성이 어떻게 사용되는지에 관한 배경 지식을 제공하는가?
- 자동화 또는 AI가 콘텐츠 제작에 유용하다고 생각한 이유를 설명하는가?

> 권장: 누군가가 '이 콘텐츠는 어떻게 만들었을까?'라고 생각할 수 있는 콘텐츠에 AI 또는 자동화 공개 문구를 사용

### 왜 이 콘텐츠가 만들어졌는가(Why)

- 사람들을 돕기 위한 콘텐츠여야 함
- 주로 검색엔진의 방문을 유도하기 위한 콘텐츠를 만들고 있다면 Google 시스템에서 보상을 제공하고자 하는 콘텐츠가 아님
- AI 생성 등의 자동화를 사용하여 검색 순위를 조작하는 것을 주목적으로 콘텐츠를 생성하는 행동은 스팸 정책 위반

# 검색엔진 최적화(SEO)가 필요한가요?

> 출처: https://developers.google.com/search/docs/fundamentals/do-i-need-seo?hl=ko

## 개요

SEO는 '검색엔진 최적화(Search Engine Optimization)' 또는 '검색엔진 최적화 전문업체(Search Engine Optimizer)'의 약어입니다. 전문가의 도움은 시간을 절약할 수 있지만 위험성도 존재합니다.

---

## SEO 전문업체의 유용한 서비스

- 사이트 콘텐츠 또는 구조 검토
- 호스팅, 리디렉션, 오류 페이지, 자바스크립트 사용에 대한 기술적 조언
- 콘텐츠 개발
- 온라인 비즈니스 개발 캠페인 관리
- 키워드 리서치
- SEO 교육
- 특정 시장과 지역에 관한 전문적 지식 제공

> 중요: Google을 통해 광고를 게재해도 Google 검색결과에서의 사이트 게재에는 영향을 주지 않습니다. 자연 검색결과 게재에는 비용이 들지 않습니다.

---

## SEO 시작하기

소규모 지역 비즈니스를 운영한다면 직접 처리할 수 있습니다.

### 추천 리소스

1. Google의 온라인 비즈니스 구축 동영상 시리즈
2. 검색 Essentials — 필수 요구사항 정리
3. Google 검색 작동 방식 — 크롤링, 색인 생성, 게재 방식
4. SEO 기본 가이드 — 상세한 기능 설명

### 기대 시간

변경을 시작해서 성과가 나타날 때까지 보통 **4개월에서 1년** 정도 소요됩니다.

---

## SEO 전문업체 선택 방법

### 1단계: 권장사항 구현의 의지 확인

SEO 전문업체의 조언을 실행하는 데 시간과 노력이 필요합니다.

### 2단계: 후보 업체 인터뷰

**업체의 전문성 관련:**

- 이전 작업 사례와 성공 사례 제시
- Google 검색 Essentials 준수 여부
- 자연 검색을 보완할 온라인 마케팅 서비스 제공 여부
- 달성 가능한 성과와 예상 기간
- 동종 업계 경험 보유 여부
- 특정 지역/국가 업체 지원 경험
- 다국어 사이트 개발 경험
- 주요 SEO 기법
- 업계 경력 기간

**업체가 당신에게 물어야 할 질문:**

- 귀사를 고유하고 가치 있게 만드는 요소
- 대상 고객
- 수익 창출 방식 및 검색결과의 역할
- 다른 광고 채널
- 경쟁업체

### 3단계: 평판 확인

이전 고객에게 유용한 서비스 제공 여부, 협업 용이성, 긍정적 결과 달성 여부를 확인합니다.

### 4단계: 감사 요청

- Search Console의 읽기 전용 액세스만 제공 (쓰기 권한 금지)
- 업체는 필요한 작업과 예상 결과를 현실적으로 설명해야 함
- "변경하면 1위를 보장한다"는 말은 위험 신호

### 5단계: 최종 결정

충분한 조사와 검토 후 선택합니다.

## 주의사항

### 순위 보장은 불가능

Google 검색결과 순위 1위를 보장해 줄 수 있는 업체는 없습니다.

다음을 주장하는 업체를 피하세요:

- 순위 보장
- Google과 '특별한 관계' 보유
- '우선 등록 권한' 보유

### 의심스러운 업체 주의

도어웨이 페이지나 폐기형 도메인 같은 기만적 콘텐츠 생성 시 사이트가 Google 색인에서 완전히 삭제될 수 있습니다.

## 피해야 할 불법적 SEO 관행

### 1. 섀도(Shadow) 도메인

SEO 업체가 고객을 대신한다며 섀도 도메인을 소유하는 경우, 관계 악화 시 도메인을 경쟁사로 연결할 수 있습니다.

### 2. 도어웨이 페이지

특정 키워드에 맞게 로드되는 페이지를 사이트에 배치하여 관련성을 높인다고 주장하지만, 숨겨진 링크를 통해 다른 고객에게 연결하거나 불법 콘텐츠로 연결될 위험이 있습니다.

### 3. 링크 구매

순위를 높이기 위해 다른 사이트의 링크를 구매하는 행위는 Google 스팸 정책 위반이며, 부분 또는 전체 사이트에 직접 조치가 취해질 수 있습니다.

# 웹사이트 검색엔진 최적화 유지관리 가이드

> 출처: https://developers.google.com/search/docs/fundamentals/get-started?hl=ko

## 개요

Google 검색에 등록된 사이트의 성능을 최적화하려면 기초 항목 이상의 추가 조치가 필요합니다.

---

## Google의 사이트 크롤링 및 색인 방법 제어

### 기본 이해

크롤링/색인 생성/게재 파이프라인을 제대로 이해하지 못하면 문제를 디버그하거나 사이트의 Google 검색 동작을 예측하기 어려울 수 있습니다.

### 리소스 접근성

- Google이 크롤링해야 할 모든 리소스(이미지, CSS) 확인
- robots.txt로 차단되지 않았는지 검증
- 익명 사용자가 접근 가능한지 확인
- URL 검사 도구로 실제 렌더링 확인 권장

### Robots.txt 활용

- 크롤링 방지에 사용
- 사이트맵으로 크롤링 유도
- 중복 콘텐츠 및 중요하지 않은 리소스 차단
- **주의**: robots.txt를 색인 생성을 방지하는 메커니즘으로 사용하지 마세요
- 대신 noindex 태그나 로그인 요구사항 사용

### 사이트맵 전략

- 중요 페이지를 Google에 알리는 핵심 방법
- 업데이트 빈도 정보 제공
- 이미지, 동영상 등 비텍스트 콘텐츠 크롤링에 필수
- Google은 사이트맵 페이지를 우선순위로 크롤링
- 자주 변경되거나 링크로 검색되지 않는 페이지에 특히 중요

### 다국어/다지역 사이트 관리

- 다국어 또는 다지역 콘텐츠 포함 시 특별 조치 필요
- hreflang 사용으로 Google에 다국어 페이지 알림
- 언어 기반 콘텐츠 조정 시 Google 크롤링 영향 확인

### 페이지 및 사이트 이전

#### 단일 URL 이전

- 완전 이동 시 301 리디렉션 구현 필수
- 일시적 이동은 302 상태 코드 사용
- 맞춤 404 페이지로 사용자 경험 개선
- soft 404가 아닌 실제 404 오류 반환 필요

#### 전체 사이트 이전

- 301 리디렉션 구현
- 사이트맵 변경
- Google에 이전 알림
- 신호를 새 사이트로 전달되도록 설정

---

## 크롤링 및 색인 권장사항

### 링크 관리

- 링크를 크롤링 가능하게 설정
- 로그인 필요 링크, 신뢰할 수 없는 콘텐츠, 유료 링크에 `rel=nofollow` 사용

### 크롤링 예산 관리

- 수억 개 페이지 또는 자주 변경되는 수천만 페이지 사이트는 전체 크롤링 불가능
- 최근 업데이트 또는 중요 페이지를 사이트맵에 우선 나열
- robots.txt로 덜 중요한 페이지 숨김

### 자바스크립트 사용

- 웹사이트의 자바스크립트에 관한 Google의 권장사항을 따릅니다

### 무한 스크롤 페이지

- Google이 스크롤하는 데 문제 가능
- 페이지 나눔 버전 제공 권장

## Google에서 내 사이트를 이해하도록 돕기

### 텍스트 콘텐츠 우선

- 주요 정보를 그래픽이 아닌 텍스트로 입력

- **뉴스 사이트**: Google 게시자 센터 도움말 참고, 뉴스 사이트맵 생성

# 웹 개발자를 위한 검색엔진 최적화 가이드

> 출처: https://developers.google.com/search/docs/fundamentals/get-started-developers?hl=ko

## 개요

Google 검색에서 더 많은 관련 사용자가 콘텐츠를 발견하려면 검색 최적화가 중요합니다. 개발자가 사이트를 검색에 최적화할 때 기술적 요소를 고려해야 합니다.

---

## 사이트가 Google에 표시되는 방법

### Googlebot의 역할

Googlebot은 Google의 웹 크롤링 봇이며, 새 페이지와 업데이트된 콘텐츠를 발견합니다.

### Google이 보는 것의 한계

- Googlebot이 항상 브라우저에 표시되는 모든 것을 인식하지는 못함
- 지원하지 않는 자바스크립트 기능으로 렌더링된 콘텐츠를 감지 실패할 수 있음
- 이미지, 동영상 등 시각적 요소가 텍스트로 표현되지 않으면 인식하지 못함

---

## 링크 확인 및 크롤링성

### 크롤링 가능한 링크 구조

- `<a>` HTML 요소를 사용하여 크롤링 가능한 링크를 만들어야 함
- 모든 페이지가 검색 가능한 다른 페이지의 링크로 연결될 수 있어야 함
- 링크 텍스트나 이미지 Alt 속성에 타겟 페이지 관련 정보를 포함해야 함

### 사이트맵 활용

사이트맵을 작성하고 제출하면 Googlebot이 사이트를 더 지능적으로 크롤링할 수 있습니다.

### 자바스크립트 앱 구조

하나의 HTML 페이지만 있는 자바스크립트 애플리케이션의 경우 각 화면이나 콘텐츠마다 고유한 URL이 필요합니다.

---

## 콘텐츠 변경 시 Google 알림

- 사이트맵을 제출합니다
- Google에 URL 재크롤링을 요청합니다
- 색인 생성 문제가 지속되면 서버 로그의 오류를 확인합니다

---

## 페이지 콘텐츠 텍스트 표현

### 시각적 콘텐츠 처리

- 모든 이미지와 동영상은 텍스트 설명을 동반해야 함
- 제품 카테고리 페이지는 이미지만 있으면 부족하며, 각 항목의 텍스트 설명이 필요

### 페이지 메타데이터

- 모든 페이지에는 구체적인 제목과 메타 설명이 있어야 함
- 고유한 제목과 설명은 검색 트래픽 증가에 도움

### 마크업 방식

- 의미론적 HTML을 사용
- 플러그인(Java, Silverlight)이나 캔버스 렌더링 콘텐츠는 색인화되지 않음
- CSS `content` 속성으로 추가된 콘텐츠는 Google 검색에서 무시될 수 있음

---

## 콘텐츠의 다양한 버전 알림

- 중복 URL을 통합
- 사이트의 현지화된 버전을 Google에 알림
- AMP 페이지를 검색 가능하게 만듦

---

## Google이 보는 콘텐츠 제어

### Googlebot 차단 방법

**페이지 접근 제한:**

- 로그인한 사용자만 접근 가능하도록 설정
- 비밀번호로 페이지를 보호

**크롤링 제어:**

- robots.txt 파일을 만들어 Googlebot의 크롤링을 제어
- robots.txt는 Google에서 웹페이지를 숨기는 메커니즘이 아님

### 색인 생성 차단

- `noindex` robots 규칙을 사용
- 크롤링은 허용하되 색인 생성은 차단할 수 있음

---

## 리치 결과(Rich Results) 활용

리치 결과는 스타일, 이미지 또는 기타 상호작용 기능을 포함하여 검색결과에서 더 눈에 띄게 표시되도록 합니다.

### 구현 방법

- 페이지에 구조화된 데이터를 제공
- 페이지 의미에 관한 명확한 단서를 제공하여 Google이 더 잘 파악하도록 함

### 리치 결과의 예

- 레시피 캐러셀
- 제품 스니펫
- 동영상 결과
- FAQ 섹션
- 이벤트 정보

# Google 검색결과에 내 웹사이트를 표시하는 방법

> 출처: https://developers.google.com/search/docs/fundamentals/get-on-google?hl=ko

## 개요

Google은 색인에 추가할 사이트를 자동으로 찾습니다. 일반적으로 웹에 게시하는 것만으로 충분하지만, Google이 놓칠 수도 있습니다.

---

## 기본 체크리스트

### 1. 웹사이트가 Google에 표시되나요?

**색인 생성 확인 방법:**

Google 검색에서 `site:example.com` 형태로 검색합니다.

**Google이 사이트를 크롤링하지 못하는 주요 이유:**

1. **외부 링크 부재** — 다른 웹사이트에서 연결되지 않음 (링크 구매는 스팸 정책 위반)
2. **새 사이트** — Google이 새로운 사이트나 변경사항을 발견하는데 수주 소요
3. **불량한 사이트 구조** — HTML이 아닌 전문 기술로 빌드된 사이트, 텍스트 없는 사이트
4. **크롤링 오류** — 로그인 페이지, Google 차단 설정
5. **미발견** — Google은 수십억 개 페이지를 크롤링하므로 일부 사이트 발견 불가

**해결 방법:**

- Search Console에 사이트 추가
- Google에 중요 URL 재크롤링 요청
- 외부 사이트에서 링크 확보

### 2. 고품질 콘텐츠 제공하나요?

- 사용자 경험을 최우선으로 고려하여 사이트 구축
- 유용하고 신뢰할 수 있는 사용자 중심 콘텐츠 작성

### 3. 지역 비즈니스 표시

- Google 비즈니스 프로필로 Google 검색과 지도 전반에서 비즈니스 정보 표시 관리 (무료)

### 4. 모든 기기에서 접근 가능한가?

- 대부분의 검색이 휴대기기에서 이루어짐
- 모든 화면 크기에서 빠른 로드 및 제대로 된 표시
- Lighthouse 같은 도구로 모바일 호환성 테스트

### 5. 웹사이트 보안

- HTTPS로 연결 보호 필수

# 생성형 AI 콘텐츠에 대한 Google 검색 안내

> 출처: https://developers.google.com/search/docs/fundamentals/using-gen-ai-content?hl=ko

## 개요

생성형 AI는 주제 조사와 원본 콘텐츠 구조화에 유용하지만, 사용자에 대한 가치 창출 없이 다량의 페이지를 생성하면 Google의 스팸 정책 위반이 될 수 있습니다.

---

## 핵심 요구사항

웹사이트에서 생성형 AI 콘텐츠 사용 시 다음을 확인해야 합니다:

- 검색 Essentials의 표준 준수
- Google 스팸 정책 준수
- 확장된 콘텐츠 악용 회피
- 기본 콘텐츠 방지 (독창성, 노력, 부가가치 부족)

---

## 정확성, 품질, 관련성 중점

자동 생성 콘텐츠 작성 시 특히 주의할 메타데이터:

- `<title>` 요소
- 메타 설명 요소
- 구조화된 데이터
- 이미지 대체 텍스트

### 구조화된 데이터 요건

- 일반 가이드라인 준수
- 개별 검색 기능별 구체적 정책 준수
- 마크업 검사 및 사용 자격 요건 확인

---

## 사용자에게 맥락 제공

콘텐츠 자동 생성 시:

- 작성 방식에 대한 배경 정보 공개
- 자동화 사용 방식 설명
- 이미지 메타데이터 추가

---

## 전자상거래 특정 요구사항

Google 판매자 센터의 AI 생성 콘텐츠 정책:

- **AI 이미지**: IPTC `DigitalSourceType` `TrainedAlgorithmicMedia` 메타데이터 필수
- **제품 데이터**: AI 생성 제품명/설명 별도 지정 및 라벨링 필요
